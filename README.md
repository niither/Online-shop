## Загальний опис

Online Shop — це консольний додаток для симуляції роботи інтернет-магазину з розділеними правами доступу для адміністраторів та покупців. Проект демонструє застосування принципів об'єктно-орієнтованого програмування (OOP), патернів проектування та SOLID у розробці на C#.

### Основні можливості:

- **Для покупців**: перегляд каталогу товарів, додавання до кошика, оформлення замовлення
- **Для адміністраторів**: управління товарами, категоріями та замовленнями
- **Система автентифікації**: розділення прав доступу для різних типів користувачів
- **Збереження даних**: робота з файловою системою для персистентності даних

## Функціональність

### Функції покупця:
- Реєстрація та авторизація в системі
- Перегляд каталогу товарів за категоріями
- Пошук товарів за назвою
- Додавання товарів до кошика
- Перегляд та редагування кошика
- Оформлення замовлення
- Перегляд історії замовлень

### Функції адміністратора:
- Додавання нових товарів до каталогу
- Редагування інформації про товари
- Видалення товарів з каталогу
- Управління категоріями товарів
- Перегляд усіх замовлень
- Зміна статусу замовлень

## Архітектура проекту

Проект побудований з використанням об'єктно-орієнтованого підходу:

```
Online-shop/
│
├── User.cs             # Базовий клас користувача
├── Admin.cs            # Адміністратор (наслідування)
├── Buyer.cs            # Покупець (наслідування)
├── Product.cs          # Товар
├── Category.cs         # Категорія товарів
├── Cart.cs             # Кошик покупця
├── Order.cs            # Замовлення
│
├── Services.cs         # Сервісні методи
├── FileManager.cs      # Робота з файлами
│
├── IFileManager.cs     # Інтерфейс для файлового менеджера
│
├── Menu.cs             # Базове меню
├── AdminMenu.cs        # Меню адміністратора
├── BuyerMenu.cs        # Меню покупця
│
├── categories.xml
│
└── Program.cs          # Точка входу в програму
```

## Реалізовані принципи

### SOLID принципи:

#### **S - Single Responsibility Principle (Принцип єдиної відповідальності)**
- Кожен клас відповідає за одну конкретну частину функціональності
- `Product.cs` - відповідає тільки за товар
- `FileManager.cs` - тільки за роботу з файлами
- `Services.cs` - тільки за методи керування кошиком і категоріями

#### **O - Open/Closed Principle (Принцип відкритості/закритості)**
- Класи відкриті для розширення, але закриті для модифікації
- Базовий клас `User` можна розширювати через наслідування (`Admin`, `Buyer`)
- Додавання нових типів користувачів не вимагає зміни існуючого коду

#### **L - Liskov Substitution Principle (Принцип підстановки Барбари Лісков)**
- Об'єкти класів-нащадків (`Admin`, `Buyer`) можуть використовуватися замість базового класу `User`
- Поліморфізм забезпечує коректну роботу з різними типами користувачів

#### **I - Interface Segregation Principle (Принцип розділення інтерфейсу)**
- Інтерфейс `IFileManager` містить тільки необхідні методи для роботи з файлами
- Класи не залежать від методів, які вони не використовують

#### **D - Dependency Inversion Principle (Принцип інверсії залежностей)**
- Високорівневі модулі не залежать від низькорівневих
- Використання інтерфейсу `IFileManager` замість конкретної реалізації
- Залежність від абстракцій, а не від конкретних класів

### ООП концепції:

#### **Інкапсуляція**
- Приховування внутрішньої реалізації класів
- Використання модифікаторів доступу (private, protected, public)
- Властивості з get/set для контрольованого доступу до полів

#### **Наслідування**
- Базовий клас `User` та похідні класи `Admin` і `Buyer`
- Базовий клас `Menu` з успадкованими `AdminMenu` і `BuyerMenu`
- Повторне використання коду та розширення функціональності

#### **Поліморфізм**
- Перевизначення методів у класах-нащадках
- Можливість працювати з об'єктами через базовий тип

#### **Абстракція**
- Використання інтерфейсів для визначення контрактів
- Робота з абстрактними типами даних

### Додаткові реалізовані концепції:

#### **Події (Events)**
- Механізм подій для сповіщення про зміни в системі
- Обробка додавання товарів до кошика
- Повідомлення про оформлення замовлення

#### **Делегати (Delegates)**
- Використання делегатів для callback-функцій
- Обробка подій через делегати

#### **Колекції (Collections)**
- List для зберігання списків товарів, користувачів
- LINQ запити для фільтрації та сортування даних

#### **Обробка винятків (Exception Handling)**
- Try-catch блоки для обробки помилок

#### **Серіалізація/Десеріалізація**
- XML серіалізація для збереження даних

#### **Патерни проектування**
- **Singleton** - для єдиного екземпляру сервісів

## Встановлення та запуск

### Крок 1: Клонування репозиторію

```bash
git clone https://github.com/niither/Online-shop.git
cd Online-shop
```

### Крок 2: Відкриття проекту у Visual Studio

1. Відкрийте файл `Online-shop.sln` у Visual Studio
2. Дочекайтесь завантаження всіх NuGet пакетів
3. Перевірте, що цільова платформа встановлена правильно

### Крок 3: Налаштування

1. Переконайтесь, що папка `data/` існує для зберігання файлів
2. При першому запуску будуть створені тестові дані

### Крок 4: Запуск

- Натисніть `F5` або `Ctrl+F5` для запуску
- Або через меню: `Debug` → `Start Debugging`

### Компіляція через командний рядок:

```bash
dotnet restore
dotnet build
dotnet run
```
